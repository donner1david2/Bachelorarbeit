\chapter{Systemarchitektur}

Für das System musste eine Gesamte Architektur entwickelt werden, welche die Komponenten HB10, Nutzer (Browser) und Server miteinander verbindet. Die einzige bereits vorhandene Komponente ist der HB10, welcher allerdings noch keine Kamera enthält. In diesem Kapitel gibt wird ein grober Überblick über die 
Architektur gegeben, die Details zu den einzelnen Komponenten werden zu einem späterem Zeitpunkt erläutert. 

\begin{figure}[H]

Der HB10 kann seine aufgenommenen Videos mittels HTTP an den Server übertragen. Als Server dient ein einfacher Webserver, welcher zugriff auf eine Datenbank und das Dateisystem hat, um empfangene Videos und deren Metadaten zu speichern. Der Nutzer erhält über den Webserver zugriff auf eine Webplattform, 
über welche er die Videos ansehen kann.

Der Dart ist Dafür zuständig, den Spielbetrieb durchzuführen, und die Videos für die Dart-Würfe aufzunehmen, diese zu komprimieren und an den Server zu senden. Der Server Speichert die Videos, mit seinen Metadaten ab, und bietet die Videos und eine Webseite für das anschauen und finden der Videos für Nutzer des Dart.
Der Nutzer erhält über einen Browser zugriff auf eine Webseite die die Metadaten der Videos vom Server anfordert damit kann der Nutzer dann sein gewünschtes Video finden, und anschließend auf der Seite anschauen.
